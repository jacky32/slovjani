validates_uniqueness_of()

uniqueness.php : 26

Validates that the specified attribute(s) are unique in the database. For each attribute or combination of attributes, it constructs a SQL query to count how many records exist with the same value(s).

public validates_uniqueness_of(array<string|int, mixed> $attributes) : array<string|int, mixed>

If the count is greater than 0, it means there is a violation of uniqueness, and an appropriate error message is added to the list of caught exceptions. The method handles both single attribute uniqueness and composite key uniqueness. For composite keys, it generates a condition that checks for the combination of attributes. It also ensures that when updating an existing record it does not count the record itself as a violation by adding a condition to exclude the current record based on its primary key(s).

Parameters
$attributes : array<string|int, mixed>

An array of attribute names or an array of arrays for composite keys that should be validated for uniqueness. Each element can be a string (for single attribute uniqueness) or an array of strings (for composite key uniqueness). For example:

  • ['email'] to validate that the email attribute is unique.
  • [['user_id', 'question_id']] to validate that the combination of user_id and question_id is unique.
  • ['email', 'username'] to validate that both email and username are unique individually.
Return values
array<string|int, mixed> —

An array of caught exceptions, where each exception is an associative array containing the class name, attribute name(s), and error message for each uniqueness violation found during validation.

private addViolationToExceptions(array<string|int, mixed> &$caught_exceptions, array<string|int, mixed>|string $attribute, array<string|int, mixed> $row) : void

If the attribute is an array, it assumes a composite unique key and adds a message indicating that the combination of attributes must be unique. If it's a single attribute, it adds a message indicating that the attribute must be unique.

Parameters

Reference to the array where caught exceptions are stored. This array will be modified if a uniqueness violation is found.

$attribute : array<string|int, mixed>|string

The attribute(s) that were validated for uniqueness. Can be a single attribute name or an array of attribute names for composite keys.

$row : array<string|int, mixed>

The result row from the uniqueness query, which contains the count of records that match the uniqueness condition. If the count is greater than 0, it indicates a violation of uniqueness.

private getIdCondition() : string

If the record has an 'id' attribute, it will exclude that id. If the model defines a composite primary key, it will exclude the combination of those keys. This ensures that when updating an existing record, it won't fail the uniqueness validation against itself.
